3.5 Exercício Prático: Montando a página inicial com os componentes (Juntando as peças do quebra-cabeça)

Módulo 3: Desenvolvimento dos Componentes com Bootstrap -

Agora que temos nossos componentes de cabeçalho (header.html), rodapé (footer.html) e card (card.html) estilizados, vamos juntá-los para montar a página inicial do EmpreGO!. É como montar um quebra-cabeça, encaixando cada peça no seu lugar.

Passo a passo:

  1. Adaptar o index.html:
{% extends "base.html" %}

{% block title %}EmpreGO! - Página Inicial{% endblock %}

{% block content %}

    <div class="container">
        <div class="row">
            <div class="col-12">
                <h1>Encontre as melhores vagas!</h1>
                <p>Aqui você encontra as vagas mais recentes das melhores empresas.</p>
            </div>
        </div>
        <div class="row mt-4">

            {% for vaga in vagas %}
                <div class="col-12 col-md-6 col-lg-4">
                    {% include 'card.html' %}
                </div>
            {% endfor %}
        </div>

    </div>

{% endblock %}

Explicação:

  • {% extends "base.html" %}: Herda do template base, que já inclui o cabeçalho e o rodapé.
  • {% block content %}: Define o conteúdo da página inicial.
  • {% for vaga in vagas %}: Itera sobre a lista de vagas (que você deve passar do seu código Python).
  • {% include 'card.html' %}: Inclui o componente de card para cada vaga.
  • Usamos as classes de grid col-12 col-md-6 col-lg-4 para criar um layout responsivo com os cards:
    • Telas pequenas: 1 card por linha (tela inteira).
    • Telas médias: 2 cards por linha (metade da tela cada).
    • Telas grandes: 3 cards por linha (1/3 da tela cada).

Ajustando a conexão para usar um DictCursor

Depois de inserir as empresas e vagas para testes, precisaremos fazer uma pequena alteração em nosso código. Atualmente, a conexão e consulta com o banco de dados retorna os dados em forma de lista, sendo necessário acessar os dados pela sua posição (índice) na variável.

Ex: vaga[1] = id_vaga

Podemos alterar nosso código para que o resultado do banco de dados seja retornado em forma de objeto com chave-valor, sendo assim, podemos acessar o valor através do nome do campo (coluna) no banco de dados.

Ex: vaga.id_vaga ou vaga.nome_empresa

Para resolver isso, você pode modificar a conexão ou a execução da consulta para retornar os resultados como dicionários (objetos com chaves nomeadas). No MySQL, isso pode ser feito configurando o cursor para utilizar o formato de dicionário.

Abra o arquivo db_functions.py e altere a seguinte linha:

import mysql.connector
from config import *

def conectar_db():
    conexao = mysql.connector.connect(
        host=DB_HOST,
        user=DB_USER,
        password=DB_PASSWORD,
        database=DB_NAME
    )
    # Alterar a linha abaixo
    cursor = conexao.cursor(dictionary=True)
    return conexao, cursor


def encerrar_db(cursor, conexao):
    cursor.close()
    conexao.close()

Agora que os dados são retornados como dicionários, você pode se referir a eles diretamente pelos nomes dos campos, como no card.html:

<div class="card mb-4 shadow">
    <div class="card-body">
        <h5 class="card-title text-primary">
            {{ vaga.titulo }}
        </h5>
        <h6 class="card-subtitle mb-2 text-muted">
            {{ vaga.nome_empresa }}
        </h6>
        <p class="card-text">
            {{ vaga.descricao | truncate(100) }}
        </p>  
        <p class="card-text">
            <small class="text-muted">
                {{ vaga.formato }} - {{ vaga.tipo }}
            </small></p>
        <a href="/sobrevaga/{{ vaga.id_vaga }}" class="btn btn-primary">
            Ver mais
        </a>
    </div>
</div>

Conclusão

Ao configurar o cursor para usar o DictCursor, você conseguirá acessar os campos retornados da consulta SQL diretamente pelos nomes das colunas, em vez de usar índices numéricos, tornando seu código mais legível e menos propenso a erros.

Ajustando a rota inicial:

Agora precisamos acrescentar a busca por vagas em nossa rota inicial para que as vagas apareçam nos cards. Abra o arquivo app.py e altere a rota inicial (/)

@app.route('/')
def index():
    if session:
        if 'adm' in session:
            login = 'adm'
        else:
            login = 'empresa'
    else:
        login = False

    try:
        comandoSQL = '''
        SELECT vaga.*, empresa.nome_empresa 
        FROM vaga 
        JOIN empresa ON vaga.id_empresa = empresa.id_empresa
        WHERE vaga.status = 'ativa'
        ORDER BY vaga.id_vaga DESC;
        '''
        conexao, cursor = conectar_db()
        cursor.execute(comandoSQL)
        vagas = cursor.fetchall()
        return render_template('index.html', vagas=vagas, login=login)
    except Error as erro:
        return f"ERRO! Erro de Banco de Dados: {erro}"
    except Exception as erro:
        return f"ERRO! Outros erros: {erro}"
    finally:
        encerrar_db(cursor, conexao)

Explicação:

  • A função index() busca as vagas no banco de dados usando a função conectar_db().
  • Passamos a lista de vagas e o status de login para o template index.html.
  • O template itera sobre vagas e exibe cada vaga usando o componente card.html.
